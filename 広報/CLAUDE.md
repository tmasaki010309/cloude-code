# 広報

## 役割
発信・ブランド管理担当。SNS投稿・ブログ記事・プロフィールなど対外的なコンテンツを管理する。

## フォルダ構成
```
広報/
├── 読み込み資料/
│   ├── ブランド素材/ ← 自己紹介・実績・サービス内容・キーワードを格納
│   └── 参考記事/    ← 参考にしたい記事・競合の発信・トレンド情報を格納
└── 出力/
    ├── SNS投稿/   ← X（Twitter）・Instagram・LinkedInなどの投稿文
    ├── ブログ記事/ ← note・自社サイト向けブログ記事
    └── プロフィール/ ← 各媒体向けのプロフィール文
```

## 処理の流れ
1. `読み込み資料/ブランド素材/` に伝えたい情報・実績を格納する
2. Claude Code に「広報：SNS投稿を作成して」と指示する
3. コンテンツが `出力/` の各フォルダに出力される

## 出力ファイル
- `出力/SNS投稿/SNS投稿_[日付].md` ― 複数プラットフォーム向け投稿文セット
- `出力/ブログ記事/[タイトル]_[日付].md` ― 公開用ブログ記事
- `出力/プロフィール/プロフィール_[媒体名].md` ― 媒体別プロフィール文

## コンテンツ作成の方針
- 専門性が伝わる内容を優先する
- 売り込みではなく「価値提供」の視点で書く
- 個人事業主としての人柄・信頼感が伝わるトーンにする

## 処理指示の例
- 「広報：[テーマ]についてのSNS投稿を3パターン作成してください」
- 「広報：[サービス名]のブログ記事を作成してください」
- 「広報：note用のプロフィールを更新してください」
