# 営業

## 役割
受注・提案管理担当。顧客情報・商談メモをもとに提案書作成・フォローアップ管理・売上把握を行う。

## フォルダ構成
```
営業/
├── 読み込み資料/
│   ├── 顧客情報/   ← 顧客リスト・企業情報・過去のやりとりメモを格納
│   └── 商談メモ/  ← 商談・打ち合わせの記録を格納
└── 出力/
    ├── 提案書/         ← 顧客向け提案書・見積書
    ├── 売上管理/       ← 案件一覧・売上推移・入金状況
    └── フォローアップ/ ← 次のアクション・連絡タイミング一覧
```

## 処理の流れ
1. `読み込み資料/顧客情報/` に顧客データを格納する
2. `読み込み資料/商談メモ/` に商談記録を格納する
3. Claude Code に「営業：[顧客名]の提案書を作成して」と指示する
4. 成果物が `出力/` の各フォルダに出力される

## 出力ファイル
- `出力/提案書/提案書_[顧客名]_[日付].md` ― 顧客向け提案書
- `出力/売上管理/案件一覧.csv` ― 全案件の進捗・金額・入金状況
- `出力/売上管理/売上サマリー_[年月].md` ― 月次売上レポート
- `出力/フォローアップ/フォローアップリスト_[日付].md` ― 次のアクション一覧

## 案件ステータス
- **リード**：接触あり、まだ提案前
- **提案中**：提案書を送付済み
- **交渉中**：条件調整中
- **受注**：契約完了
- **失注**：今回は見送り

## 処理指示の例
- 「営業：[顧客名]の商談メモを読んで提案書を作成してください」
- 「営業：今月の売上サマリーを作成してください」
- 「営業：フォローアップが必要な案件を一覧にしてください」
