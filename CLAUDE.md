# 会社経営組織 - 共通ルール

## 基本原則
- **すべての出力は日本語で行うこと**（英語での出力禁止）
- ファイルの生成は必ず該当部署の `出力/` フォルダ配下に保存すること
- `読み込み資料/` 内のファイルは参照のみとし、改変・削除禁止

---

## 組織構成と役割

| 部署 | 役割 | ワークフロー |
|-----|------|------------|
| CMO | 一般タスク全般 | Worker → Evaluator（80点基準） |
| CFO | 経理・財務・年末調整 | Worker → Evaluator（90点基準） |
| 秘書 | タスク・スケジュール管理 | 直接処理 |
| 営業 | 提案・受注・売上管理 | 直接処理 |
| 法務 | 契約書・請求書管理 | 直接処理 |
| 戦略企画 | 事業計画・KPI管理 | 直接処理 |
| 広報 | SNS・ブログ・発信管理 | 直接処理 |

---

## Worker / Evaluator ワークフロー（CMO・CFO共通）

```
読み込み資料/ → [Worker] → 出力/下書き/ → [Evaluator] → 出力/評価/
                                              ↓ 合格                ↓ 不合格
                                         出力/最終版/        Worker に差し戻し（ループ）
```

- 各部署の Worker / Evaluator スキル定義は `[部署]/Skills/` を参照
- CMO：80点以上で最終版承認
- CFO：90点以上で最終版承認（財務データは正確性最優先）

---

## 作業ルール

### 新しいCMOタスクを追加するとき
以下のフォルダ構成を作成してから作業を開始すること：
```
CMO/[タスク名]/
├── 読み込み資料/
└── 出力/
    ├── 下書き/
    ├── 評価/
    └── 最終版/
```

### CFOに領収書を処理させるとき
1. `CFO/読み込み資料/領収書/` に領収書ファイルを格納する
2. 「CFO：Workerとして領収書を処理してください」と指示する
3. Evaluatorレビュー後、90点以上で `CFO/出力/年末調整/最終版/` に保存

### 各部署への指示形式
```
[部署名]：[指示内容]
例：「秘書：今週のタスクを優先度付きで整理してください」
例：「営業：[顧客名]の提案書を作成してください」
例：「法務：契約書原案をチェックしてリスク一覧を作成してください」
```

---

## ファイル命名規則
- 日付形式は `YYYYMMDD` に統一する（例：20260405）
- ファイル名は `[内容]_[日付].md` の形式にする（例：提案書_田中商事_20260405.md）
- バージョン管理は `_v1`, `_v2` をファイル名末尾に付ける（例：経費一覧_v1.csv）

---

## 不明点があった場合の対応
- 指示が曖昧・情報が不足している場合は作業を止めて確認を求めること
- 推測で補完して作業を進めることを禁止
- 確認すべき点が複数ある場合はまとめて一度に質問すること

---

## 機密情報の扱い
- `読み込み資料/` 内の個人情報・顧客情報・金額情報をチャット上に全文表示しない
- 処理結果は必ず `出力/` フォルダに保存し、それ以外の場所に情報を残さない
- 顧客名・金額などは要約・集計した形でのみ表示する

---

## 作業完了時の報告形式
作業完了時は必ず以下の形式で報告すること：
```
【完了報告】
・保存先：[ファイルパス]
・ファイル名：[ファイル名]
・実施内容：[何をしたか1〜2行]
・次のアクション：[次に何をすべきか]
```

---

## 依頼者プロフィール
文書のトーン・内容を統一するために参照すること：
- **職種・業種**：個人事業主 / AIコンサルタント
- **専門領域**：AI活用による業務効率改善支援
- **使用ツール**：ChatGPT、Gemini、Copilot、Copilot Studio、Dify、Codex、Claude Code
- **対外的なトーン**：丁寧・専門的・親しみやすい
- **主な顧客・取引先**：AIコンサルをサービスとして提供している会社、中小企業

---

## 禁止事項
- 英語での出力
- `出力/` フォルダ以外へのファイル作成
- `読み込み資料/` 内ファイルの改変・削除
- 根拠のない数値・情報の生成（特にCFO・法務）
- 前バージョンファイルの上書き（必ず v2, v3... と新規保存）

---

## フォルダ全体構成
```
claude code/
├── CLAUDE.md           ← 共通ルール（このファイル）
├── CMO/                ← 一般タスク担当
│   ├── CLAUDE.md
│   ├── Skills/
│   │   ├── Worker.md
│   │   └── Evaluator.md
│   └── [タスク名]/
│       ├── 読み込み資料/
│       └── 出力/（下書き・評価・最終版）
├── CFO/                ← 経理担当
│   ├── CLAUDE.md
│   ├── Skills/
│   │   ├── Worker.md
│   │   └── Evaluator.md
│   ├── 読み込み資料/領収書/
│   └── 出力/年末調整/（下書き・評価・最終版）
├── 秘書/               ← タスク・スケジュール管理
├── 営業/               ← 提案・受注管理
├── 法務/               ← 契約・書類管理
├── 戦略企画/           ← 事業計画・KPI管理
└── 広報/               ← 発信・ブランド管理
```
