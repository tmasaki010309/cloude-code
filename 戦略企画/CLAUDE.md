# 戦略企画

## 役割
事業計画・KPI管理担当。目標設定・進捗管理・振り返りを通じて事業の方向性を管理する。

## フォルダ構成
```
戦略企画/
├── 読み込み資料/
│   ├── 市場調査/    ← 業界動向・競合情報・参考資料を格納
│   └── 事業データ/ ← 売上データ・実績数値・過去の計画書を格納
└── 出力/
    ├── 事業計画/   ← 年次・四半期の事業計画書
    ├── KPIレポート/ ← 目標 vs 実績の進捗レポート
    └── 振り返り/   ← 月次・四半期の振り返りレポート
```

## 処理の流れ
1. `読み込み資料/事業データ/` に実績データを格納する
2. Claude Code に「戦略企画：KPIレポートを作成して」と指示する
3. レポートが `出力/` の各フォルダに出力される

## 出力ファイル
- `出力/事業計画/事業計画_[年度].md` ― 年次・四半期事業計画書
- `出力/KPIレポート/KPIレポート_[年月].md` ― 月次KPI進捗レポート
- `出力/振り返り/振り返り_[年月].md` ― 月次・四半期振り返りレポート

## KPI管理の観点
- **売上**：月次売上・目標比・前月比・前年比
- **案件**：新規問い合わせ数・受注率・平均単価
- **活動**：稼働時間・タスク完了率・発信回数
- **財務**：経費率・利益率・キャッシュフロー

## 処理指示の例
- 「戦略企画：今月のKPIレポートを作成してください」
- 「戦略企画：来期の事業計画書を作成してください」
- 「戦略企画：四半期振り返りをまとめてください」
